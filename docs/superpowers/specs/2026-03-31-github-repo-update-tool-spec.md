# GitHub 仓库本地更新工具 - 需求规格说明书

**文档版本:** v1.0  
**创建日期:** 2026-03-31  
**作者:** 需求与架构师

## 1. 项目概述

### 1.1 项目背景

个人开发者在本地维护多个 GitHub 仓库时，需要批量检查和更新这些仓库。手动逐个检查和更新效率低下，容易遗漏。

### 1.2 项目目标

开发一个自动化工具，能够：
- 批量检查本地 GitHub 仓库是否有新更新
- 自动拉取更新
- 记录更新历史到 CSV 文件

### 1.3 目标用户

- 个人开发者（主要）
- 需要同步本地代码仓库的团队

## 2. 功能需求

### 2.1 核心功能

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|--------|
| F01 | 接收仓库路径 | 接受一个或多个本地仓库路径作为输入参数 | P0 |
| F02 | 检查更新 | 使用 `git pull --dry-run` 检查仓库是否有新更新 | P0 |
| F03 | 拉取更新 | 如果有新更新，执行 `git pull` 更新仓库 | P0 |
| F04 | 记录更新信息 | 将仓库名、本地路径、更新时间、更新履历记录到 CSV 文件 | P0 |

### 2.2 功能详情

#### F01：接收仓库路径

**输入方式:** 命令行参数

**参数格式:**
- 支持绝对路径：`D:\repo1`、`/home/user/repo2`
- 支持相对路径：`../repo3`、`./repo4`
- 支持多个路径：`"path1" "path2" "path3"`

**验证规则:**
- 参数数量：至少 1 个
- 路径存在性：路径必须存在
- Git 仓库验证：必须包含 `.git` 目录

**错误处理:**
- 未提供参数：打印使用提示，退出程序
- 路径不存在：记录错误，继续处理其他仓库

#### F02：检查更新

**技术实现:**
- 使用 Python `subprocess` 模块执行命令
- 命令：`git pull --dry-run`

**返回值:**
- True：有新更新
- False：无更新
- Exception：执行失败

**输出解析:**
- 解析命令输出，判断是否有可拉取的提交

#### F03：拉取更新

**技术实现:**
- 使用 Python `subprocess` 模块执行命令
- 命令：`git pull`

**输出处理:**
- 捕获命令输出
- 格式化为更新履历（替换换行符为空格）

#### F04：记录更新信息

**文件命名:**
- 格式：`result_YYYYMMDD.csv`
- 示例：`result_20260331.csv`
- 位置：工具运行目录

**CSV 列结构:**

| 列名 | 说明 | 示例 |
|------|------|------|
| 仓库名称 | 仓库名（路径最后一部分） | repo1 |
| 本地路径 | 完整本地路径 | D:\repo1 |
| 更新时间 | 格式化时间戳 | 2026-03-31 10:30:15 |
| 更新履历 | 更新详情或状态信息 | fast-forward: ... |

**记录策略:**
- 追加模式：每次运行在文件末尾追加新记录
- 所有仓库（包括无更新和失败）都记录
- 首次写入时添加表头

### 2.3 非功能需求

| 需求类型 | 具体要求 |
|---------|---------|
| 性能 | 单个仓库处理时间 < 5 秒 |
| 可靠性 | 单个仓库失败不影响其他仓库处理 |
| 可维护性 | 代码模块化，职责清晰 |
| 易用性 | 提供详细的控制台输出和错误提示 |

## 3. 架构设计

### 3.1 整体架构

采用双层架构设计：

```
┌─────────────┐
│  用户界面层  │
│  .bat 文件   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  业务逻辑层  │
│ Python 脚本 │
└──────┬──────┘
       │
       ├─────────┐
       ▼         ▼
┌──────────┐ ┌──────────┐
│ Git 命令 │ │ CSV 文件 │
└──────────┘ └──────────┘
```

**设计理念:**
- 分离用户交互和业务逻辑
- 便于后续维护和跨平台支持

### 3.2 模块划分

| 模块 | 职责 | 依赖 |
|------|------|------|
| main | 程序入口，参数解析，流程协调 | 其他所有模块 |
| git_ops | Git 命令执行与结果解析 | subprocess |
| csv_writer | CSV 文件读写 | csv, datetime |
| utils | 工具函数（路径验证、名称提取等） | os, re |

## 4. 组件设计

### 4.1 Python 脚本（update_github_repos.py）

#### 文件结构

```
update_github_repos.py
├── 导入模块
├── 常量定义
├── 工具函数
│   ├── validate_repo_path()
│   ├── extract_repo_name()
│   ├── get_current_time()
│   └── format_update_message()
├── 业务函数
│   ├── check_repo_update()
│   ├── pull_repo()
│   └── log_update_result()
├── 主函数
│   └── main()
└── 程序入口
    └── if __name__ == '__main__'
```

#### 核心函数定义

##### main(args)

**功能:** 程序主入口

**参数:**
- args: 命令行参数列表

**返回值:**
- 0: 成功
- 1: 失败

**流程:**
1. 解析命令行参数
2. 验证参数有效性
3. 遍历处理每个仓库
4. 输出统计信息
5. 返回退出码

##### validate_repo_path(repo_path)

**功能:** 验证路径是否为有效的 Git 仓库

**参数:**
- repo_path: 仓库路径（字符串）

**返回值:**
- True: 有效 Git 仓库
- False: 无效

**验证步骤:**
1. 检查路径是否存在
2. 检查是否为目录
3. 检查 `.git` 子目录是否存在

##### extract_repo_name(repo_path)

**功能:** 从完整路径提取仓库名称

**参数:**
- repo_path: 仓库路径（字符串）

**返回值:**
- 仓库名称（字符串）

**示例:**
- `D:\code\repo1` → `repo1`
- `/home/user/repo2` → `repo2`
- `C:\Users\user\projects\my-repo` → `my-repo`

##### check_repo_update(repo_path)

**功能:** 检查仓库是否有新更新

**参数:**
- repo_path: 仓库路径（字符串）

**返回值:**
- True: 有更新
- False: 无更新
- None: 执行失败

##### pull_repo(repo_path)

**功能:** 执行 git pull 更新仓库

**参数:**
- repo_path: 仓库路径（字符串）

**返回值:**
- 成功：更新履历字符串
- 失败：None

##### log_update_result(repo_name, repo_path, update_info, csv_path)

**功能:** 将更新记录写入 CSV 文件

**参数:**
- repo_name: 仓库名称（字符串）
- repo_path: 本地路径（字符串）
- update_info: 更新履历或错误信息（字符串）
- csv_path: CSV 文件路径（字符串）

### 4.2 批处理文件（update_repos.bat）

#### 文件内容

```batch
@echo off
chcp 65001
echo =====================================
echo   GitHub 仓库本地更新工具
echo =====================================
echo.

python update_github_repos.py %*

if %errorlevel% neq 0 (
    echo.
    echo 程序执行失败，错误码: %errorlevel%
) else (
    echo.
    echo 程序执行成功
)

echo.
pause
```

#### 功能说明

- 设置控制台编码为 UTF-8（支持中文）
- 调用 Python 脚本，传递所有参数
- 显示执行结果
- 暂停等待用户按键（避免窗口关闭过快）

## 5. 数据流设计

### 5.1 主流程图

```
开始
  ↓
解析命令行参数（仓库路径列表）
  ↓
初始化计数器（成功、失败、总计）
  ↓
遍历每个仓库路径：
  ├─ 输出当前仓库信息
  ├─ 验证路径是否存在
  │   ├─ 不存在 → 记录错误，失败计数+1
  │   └─ 存在 → 继续
  ├─ 验证是否为 Git 仓库
  │   ├─ 不是 → 记录错误，失败计数+1
  │   └─ 是 → 继续
  ├─ 执行 git pull --dry-run
  │   ├─ 有更新 → 执行 git pull
  │   │            ├─ 成功 → 记录更新信息，成功计数+1
  │   │            └─ 失败 → 记录错误，失败计数+1
  │   └─ 无更新 → 记录"已是最新"，成功计数+1
  └─ 异常 → 记录错误，失败计数+1
  ↓
输出统计信息
  ↓
返回退出码
  ↓
结束
```

## 6. 错误处理

### 6.1 错误分类

| 错误类型 | 错误代码 | 描述 | 处理方式 |
|---------|---------|------|---------|
| E001 | 无参数 | 未提供仓库路径参数 | 打印使用提示，退出程序 |
| E002 | 路径不存在 | 指定的路径不存在 | 记录到 CSV，继续下一个仓库 |
| E003 | 非目录 | 路径不是目录 | 记录到 CSV，继续下一个仓库 |
| E004 | 非 Git 仓库 | 目录不是 Git 仓库 | 记录到 CSV，继续下一个仓库 |
| E005 | Git 命令失败 | git 命令执行失败 | 记录到 CSV，继续下一个仓库 |
| E006 | CSV 写入失败 | 无法写入 CSV 文件 | 打印警告，继续处理 |

### 6.2 错误信息格式

**控制台输出:**
```
[错误] D:\repo3 - 不是 Git 仓库
```

**CSV 记录:**
```
error: 不是 Git 仓库
```

### 6.3 异常处理策略

1. **try-except 包裹所有 Git 命令**
2. **subprocess.CalledProcessError**: 记录命令返回码和错误输出
3. **FileNotFoundError**: 记录命令未找到（Git 未安装）
4. **PermissionError**: 记录权限不足
5. **其他异常**: 记录异常类型和消息
